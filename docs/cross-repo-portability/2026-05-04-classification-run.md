The subagent was instructed to read each script's leading comment block and `main()`/CLI usage — not guess from filenames — and classify each script as `repo-local`, `cross-repo-portable`, or `host-substrate-deferred`. ## Codebase context (from the agent's own report)

- Two distinct substrate families dominate. **Substrate-A** is per-task: `EXPERIMENT.yaml` at the repo root, parsed via `@minsky/experiment-record` (`scripts/check-pivot-success-margin.mjs:67`, `scripts/check-anchor-primary-source.mjs:59`, `scripts/check-measurement-inspects-output.mjs:56`). **Substrate-B** is per-PR: PR body text fed in via stdin or a path arg (`scripts/check-pr-self-grade.mjs:107-119`).
- The remaining scripts walk minsky-internal trees: `novel/**` (`scripts/check-rule-1-novel-justification.mjs:22`, `scripts/check-rule-4-otel-coverage.mjs:5`, `scripts/check-rule-6-let-it-crash.mjs:26`), `vision.md` § "Pattern conformance index" (`scripts/check-pattern-index.mjs:7-13`), `vision.md` Glossary (`scripts/check-rule-5-glossary-discipline.mjs:9-21`), `ARCHITECTURE.md` dependency table (`scripts/check-rule-2-dep-coverage.mjs:3-7`), `novel/spec-monitor/SKILL.md` (`scripts/check-skill-rule-cap.mjs:50`), and `novel/**/README.md` failure-mode tables (`scripts/check-rule-7-chaos-coverage.mjs:4-19`).

## Recommendations (subagent's own)

1. **Highest leverage when ported is the `EXPERIMENT.yaml` triplet** (anchor-primary-source, measurement-inspects-output, pivot-success-margin) plus pr-self-grade. All four operate on per-task substrate that travels with the runner, not on host-repo trees. If the runner ever does ship a constitutional gate, *these four are the gate* — they catch the highest-impact failure modes (vanity success thresholds, non-inspecting measurements, blog-decay anchors, hypothesis-without-self-grade) on every task in any repo.
2. **Dead-weight for cross-repo: rule-1, rule-2, rule-7, skill-rule-cap.** They're tied to minsky-coined artefacts (`research.md`, `ARCHITECTURE.md` dep table, per-package README chaos tables, `spec-monitor/SKILL.md`). Don't try to overlay; let them stay in `fyodoriv/minsky` where they belong.
3. **rule-6 (let-it-crash) is the one "deferrable" rule** — the AST shape check is generic, but every credible host repo already has equivalent ESLint coverage. Declare the host's `npm run verify` / `yarn run -T eslint` as the substrate and skip re-running this lint cross-repo.
4. **The host overlay only needs to drop two files** to enable the 4 portable rules: `EXPERIMENT.yaml` at the worktree root (the per-task hypothesis record) and a PR-body file that the runner produces from `gh pr view --json body`. No host-repo source-tree modification is required for the portable set.
5. **Make the diff-base override consistent before shipping.** `RULE_3_DIFF_BASE` env var (`check-rule-3-doc-first.mjs:199`), `PATTERN_INDEX_DIFF_BASE` (`check-pattern-index.mjs:28`), and `--diff-base=<ref>` flags exist piecemeal. If the runner ever wants to reuse any of these for a host like `iep-capabilities-3` (which uses `master`, not `main`), normalize on a single `MINSKY_DIFF_BASE` env var across all 13 scripts in the same PR. Low-risk refactor, large portability win.

## How this run was rebutted

Two hours after this run, the operator's vision-clarification — "the whole minsky has to work on other repos with all rules; it's OK to create globally git-ignored files in the host" — flipped the framing: the host *can* have minsky's substrate, just gitignored from the host's git history. Under that new framing, 8 of the 9 repo-local rules become `sidecar-portable` (the substrate is path-shaped, the path becomes parameter-isable via `MINSKY_HOST_ROOT`), and the aggregate flips to 12/13 = 92.3 % portable. Only `check-rule-1-novel-justification` stays repo-local *by design* — the host's source taxonomy is the host's domain, not minsky's. See [`../cross-repo-portability.md`](../cross-repo-portability.md) for the rebutted version.

The pivot the naive run would have triggered (thin-envoy mode) is therefore **not** triggered under the operative architecture. Recorded here because rule-#9 pre-registration requires the rejected-pivot path to be visible: future quarterly reviews comparing predicted-vs-observed must see both numbers.
